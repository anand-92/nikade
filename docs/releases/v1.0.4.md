# openOwl v1.0.4 Release Notes

**Release Date**: 2026-03-23

Security fix + terminal scrollbar + ESC key + TCC prompt elimination + startup performance + macOS Tahoe design compliance. Three rounds of parallel code review (9 agents).

---

## Critical Fixes

### Deployment removeItem Deleting User Directories

`DeploymentStore.removeDeployment` and `cloneRepo` used `FileManager.removeItem` without path validation. If `clonePath` was empty or pointed outside `~/.openowl/deployments/`, the recursive delete could destroy user project directories.

**Fix:** Added `isSafeDeploymentPath` guard — only allows deletion within `~/.openowl/deployments/`. Unsafe paths refused with log warning or thrown error.

### Path Prefix Bypass (Code Review R2)

`isSafeDeploymentPath` used `hasPrefix` without trailing `/`, allowing sibling directories like `.openowl/deployments-evil` to pass the check.

**Fix:** Append `/` to prefix: `safePrefix + "/"`. Added test case for sibling directory name attack.

### removeItem Error Silently Swallowed (Code Review R2)

`try? FileManager.default.removeItem` in `removeDeployment` discarded file deletion errors — user saw success but orphaned files remained on disk.

**Fix:** `try?` → `do/try/catch` with `NSLog` on failure.

## Terminal Fixes

### ESC Key Not Working

ESC never reached the terminal. `NavigationSplitView` consumed ESC via `performKeyEquivalent` (for sidebar collapse) before `TerminalNSView.keyDown` could process it.

**Fix:** Claim ESC (plain and Shift+ESC) in `TerminalNSView.performKeyEquivalent`, forward to `keyDown`. Guarded by `firstResponder === self` so ESC in search/commit fields is unaffected.

### Scrollbar Implementation

Terminal had no visible scrollbar. Root causes:
1. ghostty 1.3.2 doesn't send `CELL_SIZE` action → document height calculation failed
2. NSScrollView's overlay scroller requires scroll activity detection, but `scrollWheel` events bypass NSScrollView (go directly to ghostty)
3. NSScrollView's tall document mode consumes scrollWheel events, fighting with ghostty's scroll handling

**Fix:** Removed NSScrollView entirely. Replaced with standalone `ScrollIndicatorView` — a custom-drawn rounded bar overlay:
- Knob size proportional to viewport/total ratio, clamped to [0, 1]
- Position tracks ghostty's scroll offset via `SCROLLBAR` action
- Uses `NSColor.labelColor` for light/dark mode adaptation
- Fades in on scroll (offset change), fades out after 1s idle
- Timer cleanup in `deinit` prevents fire-after-dealloc
- No event conflicts — scrollWheel goes directly to TerminalNSView → ghostty

### Editor Tab Obscured by SourceEditor

SourceEditor content (line numbers) overflowed its frame and covered the editor tab bar above it.

**Fix:** Added `.clipped()` to SourceEditor view.

## TCC Fix

### Eliminated Repeated Permission Prompts

macOS TCC prompted "openOwl needs access to your files" on every dev build launch. Root cause: `changeCurrentDirectoryPath` calls `chdir()` on the app process, triggering TCC checks. Dev builds get re-signed → TCC treats as new app.

**Fix:** Follow Ghostty's approach — pass working directory directly to `ghostty_surface_config_s.working_directory` when creating each surface. Removed all `changeCurrentDirectoryPath` calls. Added `NSLog` when `initialWorkingDirectory` is nil (fallback to process cwd) for diagnostics.

## Performance

### Deployment Recovery Blocking App Launch

`recoverRunningDeployments` mutated `@Observable` array in a loop, triggering N separate re-renders.

**Fix:** Batch all status changes into a local copy, assign once. Health checks moved to async `Task`. Dead process transitions now logged with `NSLog`.

## UI Fixes

### Git Changes Panel Width (macOS 26)

Changes panel didn't fill HSplitView width on macOS 26.

**Fix:** Added `.frame(maxWidth: .infinity)` to changesPanel VStack.

## Design System: macOS Tahoe (26) Compliance

Comprehensive design specification compliance pass — 15 files, 231 insertions, 166 deletions.

### AppFonts Semantic Migration (155 → 4 hardcoded)

All `AppFonts` constants converted to computed properties with `#available(macOS 26, *)` branching. On macOS 26+ they return semantic text styles (`.headline`, `.callout`, `.subheadline`, `.caption`) that scale with Dynamic Type; on macOS 14 they keep fixed sizes unchanged.

- 108 direct `.font(.system(size:))` calls across 12 files migrated to `AppFonts` constants
- New diff-specific font: `diffCode` (fixed-size monospaced, alignment-critical)
- New icon sizing: `toolbarIcon`, `smallIcon`, `tinyIcon` (SF Symbol sizing in buttons)
- 4 intentional exceptions: QuickOpenSheet search field (size 13/14), attributed string highlight (dynamic weight), DeploymentRow 7pt status dot
- Duplicate constants aliased: `number = value`, `character = string`

### Color Consolidation

- New `AppEditorTheme` enum: 11 syntax highlighting colors centralized from inline `NSColor(calibratedRed:)` in FileExplorerView and GitChangesView
- New `AppGraphColors` enum: 10 Git graph lane colors centralized from inline `Color(red:)` in GitChangesView
- 5 UI opacity colors (`Color.white/black.opacity`) replaced with semantic colors (`AppPalette.border`, `Color.primary.opacity`, `Color.secondary.opacity`)
- DeploymentPanelView log panel: hardcoded grayscale replaced with `AppPalette.base` / `AppPalette.textPrimary`

### NSVisualEffectView → SwiftUI Material

`EffectView` (NSVisualEffectView wrapper) replaced with `.background(.regularMaterial)` at both usage sites (FileExplorerView, GitChangesView). On macOS Tahoe this automatically upgrades to Liquid Glass. `EffectView.swift` marked `@available(*, deprecated)`.

### Liquid Glass New API Adoption

- `TerminalSearchOverlay`: glass effect replaces manual shadow/fill background (fallback to old style on pre-26)
- `GlassCompat.swift`: new `scrollEdgeEffectIfAvailable(for:)` helper
- `SidebarView` List + `GitChangesView` ScrollView: scroll edge effect (`.soft`, top) for content-under-glass legibility
- Total glass/scrollEdge usage: 6 → 9

### Accessibility Labels (1 → 32)

Added `.accessibilityLabel()` to all icon-only buttons across 7 files:
- GitChangesView: Refresh, Pull, Push, Stage All, Unstage All, Discard All, Generate AI message, Stop generating, Toggle file list, Dismiss (11)
- SidebarView: Open project folder, Dismiss, Create worktree, Copy path ×2, Archive worktree (6)
- FileExplorerView: Quick Open, Refresh, Dismiss, Close tab (4)
- TerminalWorkspaceView: Restore pane, Split horizontally, Split vertically (3)
- TerminalSearchOverlay: Previous match, Next match, Close (3)
- DeploymentPanelView: Copy clone path, Delete deployment, Refresh logs (3)
- DeploymentRow: dynamic Start/Stop (1)
