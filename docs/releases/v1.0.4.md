# openOwl v1.0.4 Release Notes

**Release Date**: 2026-03-23

Security fix + terminal scrollbar + ESC key + TCC prompt elimination + startup performance + macOS Tahoe design compliance.

## Critical Fix

### Deployment removeItem Deleting User Directories

`DeploymentStore.removeDeployment` and `cloneRepo` used `FileManager.removeItem` without path validation. If `clonePath` was empty or pointed outside `~/.openowl/deployments/`, the recursive delete could destroy user project directories.

**Fix:** Added `isSafeDeploymentPath` guard â€” only allows deletion within `~/.openowl/deployments/`. Unsafe paths refused with log warning or thrown error.

## Terminal Fixes

### ESC Key Not Working

ESC never reached the terminal. `NavigationSplitView` consumed ESC via `performKeyEquivalent` (for sidebar collapse) before `TerminalNSView.keyDown` could process it.

**Fix:** Claim ESC (plain and Shift+ESC) in `TerminalNSView.performKeyEquivalent`, forward to `keyDown`. Guarded by `firstResponder === self` so ESC in search/commit fields is unaffected.

### Scrollbar Implementation

Terminal had no visible scrollbar. Root causes:
1. ghostty 1.3.2 doesn't send `CELL_SIZE` action â†’ document height calculation failed
2. NSScrollView's overlay scroller requires scroll activity detection, but `scrollWheel` events bypass NSScrollView (go directly to ghostty)
3. NSScrollView's tall document mode consumes scrollWheel events, fighting with ghostty's scroll handling

**Fix:** Removed NSScrollView entirely. Replaced with standalone `ScrollIndicatorView` â€” a custom-drawn rounded bar overlay:
- Knob size proportional to viewport/total ratio
- Position tracks ghostty's scroll offset via `SCROLLBAR` action
- Fades in on scroll (offset change), fades out after 1s idle
- No event conflicts â€” scrollWheel goes directly to TerminalNSView â†’ ghostty

### Auto-Scroll to Bottom During Output

When user scrolled up to review TUI output, the view jumped back to bottom on every new line.

**Fix:** Track `userScrolledUp` flag â€” set on upward scroll, cleared when ghostty reports viewport at bottom. When set, skip forced scroll.

## TCC Fix

### Eliminated Repeated Permission Prompts

macOS TCC prompted "openOwl needs access to your files" on every dev build launch. Root cause: `changeCurrentDirectoryPath` calls `chdir()` on the app process, triggering TCC checks. Dev builds get re-signed â†’ TCC treats as new app.

**Fix:** Follow Ghostty's approach â€” pass working directory directly to `ghostty_surface_config_s.working_directory` when creating each surface. Removed all `changeCurrentDirectoryPath` calls. The app process never accesses project directories directly; child shell processes inherit authorization without prompts.

## Performance

### Deployment Recovery Blocking App Launch

`recoverRunningDeployments` mutated `@Observable` array in a loop, triggering N separate re-renders.

**Fix:** Batch all status changes into a local copy, assign once. Health checks moved to async `Task`.

## UI Fixes

### Git Changes Panel Width (macOS 26)

Changes panel didn't fill HSplitView width on macOS 26.

**Fix:** Added `.frame(maxWidth: .infinity)` to changesPanel VStack.

## Design System: macOS Tahoe (26) Compliance

Comprehensive design specification compliance pass â€” 15 files, 231 insertions, 166 deletions.

### AppFonts Semantic Migration (155 â†’ 4 hardcoded)

All `AppFonts` constants converted to computed properties with `#available(macOS 26, *)` branching. On macOS 26+ they return semantic text styles (`.headline`, `.callout`, `.subheadline`, `.caption`) that scale with Dynamic Type; on macOS 14 they keep fixed sizes unchanged.

- 108 direct `.font(.system(size:))` calls across 12 files migrated to `AppFonts` constants
- New diff-specific fonts: `diffCode`, `diffLineNo`, `diffMeta` (fixed-size monospaced, alignment-critical)
- New icon sizing: `toolbarIcon`, `smallIcon`, `tinyIcon` (SF Symbol sizing in buttons)
- 4 intentional exceptions: QuickOpenSheet search field (size 13/14), attributed string highlight (dynamic weight), DeploymentRow 7pt status dot

### Color Consolidation

- New `AppEditorTheme` enum: 11 syntax highlighting colors centralized from inline `NSColor(calibratedRed:)` in FileExplorerView and GitChangesView
- New `AppGraphColors` enum: 10 Git graph lane colors centralized from inline `Color(red:)` in GitChangesView
- 5 UI opacity colors (`Color.white/black.opacity`) replaced with semantic colors (`AppPalette.border`, `Color.primary.opacity`, `Color.secondary.opacity`)
- DeploymentPanelView log panel: hardcoded grayscale replaced with `AppPalette.base` / `AppPalette.textPrimary`

### NSVisualEffectView â†’ SwiftUI Material

`EffectView` (NSVisualEffectView wrapper) replaced with `.background(.regularMaterial)` at both usage sites (FileExplorerView, GitChangesView). On macOS Tahoe this automatically upgrades to Liquid Glass. `EffectView.swift` marked `@available(*, deprecated)`.

### Liquid Glass New API Adoption
