# openOwl v1.0.4 Release Notes

**Release Date**: 2026-03-23

Security fix + terminal scrollbar + ESC key + TCC prompt elimination + startup performance.

---

## Critical Fix

### Deployment removeItem Deleting User Directories

`DeploymentStore.removeDeployment` and `cloneRepo` used `FileManager.removeItem` without path validation. If `clonePath` was empty or pointed outside `~/.openowl/deployments/`, the recursive delete could destroy user project directories.

**Fix:** Added `isSafeDeploymentPath` guard — only allows deletion within `~/.openowl/deployments/`. Unsafe paths refused with log warning or thrown error.

## Terminal Fixes

### ESC Key Not Working

ESC never reached the terminal. `NavigationSplitView` consumed ESC via `performKeyEquivalent` (for sidebar collapse) before `TerminalNSView.keyDown` could process it.

**Fix:** Claim ESC (plain and Shift+ESC) in `TerminalNSView.performKeyEquivalent`, forward to `keyDown`. Guarded by `firstResponder === self` so ESC in search/commit fields is unaffected.

### Scrollbar Implementation

Terminal had no visible scrollbar. Root causes:
1. ghostty 1.3.2 doesn't send `CELL_SIZE` action → document height calculation failed
2. NSScrollView's overlay scroller requires scroll activity detection, but `scrollWheel` events bypass NSScrollView (go directly to ghostty)
3. NSScrollView's tall document mode consumes scrollWheel events, fighting with ghostty's scroll handling

**Fix:** Removed NSScrollView entirely. Replaced with standalone `ScrollIndicatorView` — a custom-drawn rounded bar overlay:
- Knob size proportional to viewport/total ratio
- Position tracks ghostty's scroll offset via `SCROLLBAR` action
- Fades in on scroll (offset change), fades out after 1s idle
- No event conflicts — scrollWheel goes directly to TerminalNSView → ghostty

### Auto-Scroll to Bottom During Output

When user scrolled up to review TUI output, the view jumped back to bottom on every new line.

**Fix:** Track `userScrolledUp` flag — set on upward scroll, cleared when ghostty reports viewport at bottom. When set, skip forced scroll.

## TCC Fix

### Eliminated Repeated Permission Prompts

macOS TCC prompted "openOwl needs access to your files" on every dev build launch. Root cause: `changeCurrentDirectoryPath` calls `chdir()` on the app process, triggering TCC checks. Dev builds get re-signed → TCC treats as new app.

**Fix:** Follow Ghostty's approach — pass working directory directly to `ghostty_surface_config_s.working_directory` when creating each surface. Removed all `changeCurrentDirectoryPath` calls. The app process never accesses project directories directly; child shell processes inherit authorization without prompts.

## Performance

### Deployment Recovery Blocking App Launch

`recoverRunningDeployments` mutated `@Observable` array in a loop, triggering N separate re-renders.

**Fix:** Batch all status changes into a local copy, assign once. Health checks moved to async `Task`.

## UI Fixes

### Git Changes Panel Width (macOS 26)

Changes panel didn't fill HSplitView width on macOS 26.

**Fix:** Added `.frame(maxWidth: .infinity)` to changesPanel VStack.

## Tests

Added `DeploymentRecoveryTests` (10 tests):
- 4 recovery tests: dead PID → error, stopped/error unchanged, batch dead processes
- 6 path safety tests: valid/invalid paths for `isSafeDeploymentPath`
