## 19:30 (checkpoint)

### 完成

1. **Cmd+F 无反应 bug 修复** — 根因：macOS 菜单系统在 `performKeyEquivalent` 之前拦截了 Cmd+F
   - 修复方案：将 Cmd+F 处理移到 `AppDelegate.handleLocalKeyDown`（`NSEvent.addLocalMonitorForEvents`，最高优先级）
   - 同时在 Terminal 菜单添加 "Find..." 菜单项（`menuTerminalSearch`），提升快捷键发现性

2. **搜索框内快捷键失效修复** — 根因：`handleLocalKeyDown` 即使在搜索框聚焦时也拦截 Cmd+T/W/D 等终端快捷键
   - 修复方案：添加 `guard NSApp.keyWindow?.firstResponder is TerminalNSView else { return false }` 检查
   - 非终端焦点（搜索框、commit message 等输入框）时全部放行，不拦截

3. **terminal Cmd+arrow 冲突修复** — 根因：单 pane 时 local monitor 拦截 Cmd+arrow，导致 ghostty 自身的 Cmd+arrow（如跳词）失效
   - 修复方案：只在 `isMultiPane`（多 pane）时拦截方向键做 pane 导航；单 pane 时放行给 ghostty

### 决策

- 用 `firstResponder is TerminalNSView` 判断当前焦点，而非检查搜索状态——更通用，适用于搜索框、commit message 等所有非终端输入框
- Cmd+arrow 只在多 pane 时拦截，单 pane 让 ghostty 处理——保持向后兼容
