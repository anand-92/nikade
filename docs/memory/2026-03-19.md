# 2026-03-19 工作日志

## Session 1

### 完成

**WindowServer 压垮问题根因分析和修复**（核心修复，6 文件）
- 根因：所有已激活的终端 tab 的 Metal surface 通过 `opacity(0)` 保留在 ZStack 中，持续向 WindowServer 提交 drawables，导致 WindowServer CPU 飙升
- 修复方案（双重保护）：
  - `TerminalWorkspaceView` — 只渲染 `activeTabID` 的 tab，移除 `activatedTabIDs` 状态
  - `TerminalNSView` — 添加 `setRenderingPaused`/`setSurfaceVisibility`，通过 `metalLayer.isHidden` 控制
  - `TerminalPanel` — 添加 `isVisible` 参数，在 `updateNSView` 中实时更新
  - `TerminalScrollView` — 添加 `setTerminalVisibility` 方法
  - `ContentView` — 传递 `isVisible: navigationStore.activeTab == .terminal`
- 修改文件：GhosttyTerminal.swift, GhosttyApp.swift, TerminalPanel.swift, TerminalScrollView.swift, TerminalWorkspaceView.swift, ContentView.swift

**Code Review 修复**（两个并行 review agent）
- 删除死代码 `updateVisiblePanes`（GhosttyAppManager）
- `setSurfaceVisibility` 加 guard 避免重复调用
- `makeBackingLayer` 中立即应用 `hostVisible` 状态
- `viewDidMoveToWindow` 重连时用 `hostVisible` 保护 focus 设置
- 更新 `setRenderingPaused` 注释（诚实说明 ghostty 内部仍可能渲染）

**Health check 指数退避优化**
- DeploymentStore health check 连续失败时从 30s 逐步退避到 5 分钟
- 服务恢复后自动回到 30s
- 减少无效 TCP 连接和 console 噪音
- 修改文件：DeploymentStore.swift

### 决策

- 选择 `metalLayer.isHidden` 方案而非移除 view hierarchy — 避免重新挂载时的视觉闪烁
- 最终采用"只渲染 active tab"+ `metalLayer.isHidden` 双重保护
- ghostty 内部渲染无法从 Swift 侧完全暂停（需 C API 支持），当前修复已解决 WindowServer 合成压力
- health check 退避选择指数退避而非完全停止 — 更用户友好
