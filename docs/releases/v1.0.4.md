# openOwl v1.0.4 Release Notes

**Release Date**: 2026-03-23

Security fix + terminal UX improvements: prevents accidental directory deletion, fixes ESC key and scroll behavior, improves startup performance.

---

## Critical Fix

### Deployment removeItem Deleting User Directories

`DeploymentStore.removeDeployment` and `cloneRepo` used `FileManager.removeItem` without path validation. If `clonePath` was empty or pointed outside `~/.openowl/deployments/`, the recursive delete could destroy user project directories.

**Fix:** Added `isSafeDeploymentPath` guard that verifies the target path is strictly within `~/.openowl/deployments/` before any deletion. Unsafe paths are refused with a log warning (`removeDeployment`) or thrown error (`cloneRepo`).

## Terminal Fixes

### ESC Key Not Working

ESC key events never reached the terminal. macOS dispatches `performKeyEquivalent` before `keyDown` — `NavigationSplitView` consumed ESC (for sidebar collapse) before `TerminalNSView.keyDown` could process it.

**Fix:** Claim ESC (plain and Shift+ESC) in `TerminalNSView.performKeyEquivalent`, forward to `keyDown`, return `true` to prevent NavigationSplitView from consuming it. Guarded by `firstResponder === self` so ESC in other input fields (search, commit message) is unaffected.

### Auto-Scroll to Bottom During Output

When a TUI program produced output and the user scrolled up to review, the view jumped back to the bottom on every new line. `synchronizeScrollView` unconditionally scrolled to ghostty's reported position whenever the scrollbar state updated.

**Fix:** Track `userScrolledUp` flag — set when user scrolls up via trackpad/scrollbar, cleared when ghostty reports viewport at bottom. When flag is set, `synchronizeScrollView` skips the forced scroll.

### Scrollbar Not Visible

Overlay scrollers only appear when NSScrollView detects scroll activity. Since `scrollWheel` events go directly to `TerminalNSView` (then to ghostty), the NSScrollView never saw scrolling and never showed the scroller.

**Fix:** Call `scrollView.flashScrollers()` in `updateScrollbar` when scrollable content exists (`total > len`).

## Performance

### Deployment Recovery Blocking App Launch

`recoverRunningDeployments` mutated `deployments[i].status` in a for-loop. Each mutation triggered a separate `@Observable` notification, causing N re-renders of all views observing `DeploymentStore` (including MenuBarExtra tray menu).

**Fix:** Batch all status changes into a local copy, then assign once (`deployments = updated`). Health checks moved to `Task` to run asynchronously after launch.

## UX Fixes

### Repeated TCC Permission Prompts in Dev Builds

macOS TCC ties file access authorization to the app's code signature. Dev builds get re-signed on each compile, so macOS treats them as new apps and re-prompts.

**Fix:** Guard `changeCurrentDirectoryPath` calls with `bookmarkStore.hasBookmark` check. Only change cwd when a security-scoped bookmark exists (user previously authorized via NSOpenPanel). Without a bookmark, terminal opens in `~` — no TCC prompt.

### Git Changes Panel Width on macOS 26

The changes panel (left side of Git tab) didn't fill its allocated width in HSplitView on macOS 26, leaving a gap before the diff panel.

**Fix:** Added `.frame(maxWidth: .infinity)` to the changesPanel VStack.

## Tests

Added `DeploymentRecoveryTests` (10 tests):
- 4 recovery tests: dead PID marked error, stopped/error unchanged, batch dead processes
- 6 path safety tests: valid path, deployments root, home dir, root dir, user project dir, nested path
