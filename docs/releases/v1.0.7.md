# openOwl v1.0.7 Release Notes

**Release Date**: 2026-05-03

Test build for the silent-exit investigation. This release keeps the v1.0.6 terminal/UI fixes, restores libghostty's runtime behavior to the v1.0.5 build mode, and adds quit confirmation when active terminal or local deployment work would be stopped.

---

## Critical Fixes

### Silent Exit While Work Is Running

macOS Quit paths (`Cmd+Q`, app menu Quit, Dock Quit, and tray Quit) could terminate openOwl without an app-level confirmation even when terminal commands or local deployments were still running. This looked like the app disappeared by itself because the quit path was clean from macOS's point of view.

**Fix:** `AppDelegate.applicationShouldTerminate(_:)` now checks both terminal and local deployment state before allowing termination.

- Terminal state uses `ghostty_app_needs_confirm_quit(app)` through `GhosttyAppManager.needsConfirmQuit()`.
- Local deployment state blocks quit when a local deployment is `building` or `running`.
- Cancel keeps the app and running work alive.
- Confirmed Quit terminates tracked local deployment processes and marks them stopped before the app exits.

### Tray Quit Bypassed Confirmation

The menu bar tray's `Quit openOwl` action previously stopped local deployments directly, then called `NSApp.terminate(nil)` after a delay. That made tray Quit behave differently from system Quit paths and skipped any chance to cancel.

**Fix:** Tray Quit now only calls `NSApp.terminate(nil)`, so every Quit entry point goes through the same `applicationShouldTerminate(_:)` confirmation gate.

## libghostty Build Mode

v1.0.6 changed `scripts/setup.sh` to default libghostty to `ReleaseSafe`. For this test build, the default is back to `ReleaseFast`, matching v1.0.5 runtime behavior.

`ReleaseSafe` remains available for diagnostics:

```bash
OPTIMIZE=ReleaseSafe ./scripts/setup.sh
```

The cache key still includes optimize mode, so `ReleaseFast` and `ReleaseSafe` builds do not accidentally reuse the same `GhosttyKit.xcframework`. For compatibility with the existing v1.0.5-era cache, `ReleaseFast` can also reuse the legacy unsuffixed cache directory.

## Files Changed

| File | Change |
|------|--------|
| `AppDelegate.swift` | Adds `applicationShouldTerminate(_:)` quit confirmation and stops tracked local deployments only after confirmed Quit |
| `GhosttyApp.swift` | Adds `needsConfirmQuit()` wrapper around `ghostty_app_needs_confirm_quit(app)` |
| `DeploymentStore.swift` | Adds local running/building detection and confirmed-quit cleanup for local deployments |
| `DeploymentTrayMenu.swift` | Routes tray Quit through the common app termination path |
| `scripts/setup.sh` | Defaults libghostty back to `ReleaseFast`; keeps `ReleaseSafe` opt-in and preserves optimize-aware cache keys |
| `project.yml` / `project.pbxproj` | Bumps marketing version to `1.0.7` |
