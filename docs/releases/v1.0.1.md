# openOwl v1.0.1 Release Notes

**Release Date**: 2026-03-16

openOwl v1.0.1 brings Git changes management, a full-featured file explorer, local deployment services, and numerous quality-of-life improvements to the macOS-native terminal workspace.

---

## New Features

### Git Changes Panel

View, stage, and commit changes without leaving your terminal workspace.

- **Changes list** with grouped staged/unstaged files and batch operations
- **Unified diff view** with syntax-highlighted additions and deletions
- **Branch management** — create, delete, switch branches with ahead/behind indicators
- **Remote operations** — fetch, pull, push with progress feedback
- **Discard changes** — single-file or batch discard with confirmation dialog
- **Auto-refresh** — file watcher triggers real-time status updates (300ms debounce)

### File Explorer

Browse and navigate your project files with native macOS performance.

- **NSOutlineView file tree** — recursive directory listing with directory-first sorting
- **Git status coloring** — file-level A/M/D/R/U indicators with parent directory propagation
- **Quick Open (Cmd+P)** — fuzzy search panel for fast file navigation
- **Context menus** — Reveal in Finder, Open in Terminal, Copy Path
- **File drag-to-terminal** — drag files from the tree into the terminal to paste shell-escaped paths
- **Multi-tab editor** — lightweight syntax-highlighted file preview

### Local Deployment Service

Clone, build, and run projects locally with system tray management.

- **One-click deploy** — clone from Git remote, run build + start commands
- **Process management** — start/stop/restart with graceful SIGTERM → SIGKILL shutdown
- **Live log streaming** — real-time stdout/stderr output in the deployment panel
- **Remote health monitoring** — periodic HTTP health checks with status indicators
- **System tray integration** — MenuBarExtra icon with deployment status and quick actions
- **Persistence** — deployments survive app restart, running processes are recovered by PID
- **Background mode** — closing the window keeps the app alive while deployments are running

## Improvements

### Terminal

- **Copy/Paste (Cmd+C/V)** now works correctly across split panes and tabs
- **Multi-pane awareness** — clipboard operations target the active pane, not hidden ones
- **Edit menu integration** — standard Copy/Paste/Select All menu items work with the terminal
- **Tab management** — Cmd+T (new tab), Cmd+W (close), Cmd+1-9 (switch), Cmd+D (split)
- **Pane drag-and-drop** — rearrange split panes by dragging

### UI / Layout

- **NavigationSplitView layout** — proper macOS sidebar with resizable/collapsible panels
- **Status bar** — displays current branch, file count, and deployment status
- **Per-project isolation** — each project maintains its own terminal state and working directory

## Bug Fixes

- Fixed terminal copy/paste not working due to incorrect `ghostty_surface_binding_action` length parameter
- Fixed paste going to hidden/inactive terminal panes instead of the focused one
- Fixed Cmd+V/C being intercepted by terminal views when on non-terminal tabs (Deployment, Git)
- Fixed potential crash from calling `makeFirstResponder` during `performKeyEquivalent`
- Fixed stale surface pointer in clipboard callback async completion

## Technical Details

- **Platform**: macOS (Apple Silicon + Intel)
- **Terminal engine**: libghostty 1.3.2 with Metal GPU rendering
- **Language**: Swift + SwiftUI/AppKit hybrid
- **Min macOS**: 14.0 (Sonoma)
