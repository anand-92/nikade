# openOwl v1.0.1 Release Notes

**Release Date**: 2026-03-18

Bug fix release addressing terminal split-pane drag reliability, a file explorer crash, and sidebar usability improvements.

---

## Bug Fixes

### Terminal Split-Pane Drag Reliability

Pane drag-and-drop now works consistently in all scenarios.

- **Root cause 1**: `TerminalScrollView` was intercepting drag events because `NSItemProvider(object: NSString)` produces `public.utf8-plain-text`, which matched the terminal's accepted drop types. Fixed by using a private UTType `com.openowl.terminal.pane-drag` that the terminal scroll view ignores.
- **Root cause 2**: The drop overlay was gated on `draggingPaneID != nil`, but SwiftUI's async view update caused the overlay to render after the drag had already started, making drop targets miss the first entry event. Fixed by always registering drop overlays regardless of drag state.

### File Explorer Crash (MinimapView)

Fixed a crash that occurred when opening files with certain editor themes.

- **Root cause**: `MinimapView.setTheme()` called `.brightnessComponent` directly on semantic catalog colors (e.g. `controlBackgroundColor`), which return an undefined value until converted to a concrete color space. Added `.usingColorSpace(.deviceRGB)?` guard before accessing `brightnessComponent`.
- **Dependency**: Patched in a fork of `CodeEditSourceEditor` (`sanvibyfish/CodeEditSourceEditor`, branch `fix/minimap-brightness-catalog-color`).

## Improvements

### Sidebar Split-Pane Rows

The per-pane status rows in the sidebar are now larger and easier to interact with.

- Font size: 10 → 11 pt
- Status dot: 5 → 7 px
- Vertical padding: 1 → 4 pt
- Hover highlight using `.quaternary` background
- Clicking a row now focuses the corresponding pane
- Full accessibility support (`.isButton` trait, VoiceOver label + hint)
